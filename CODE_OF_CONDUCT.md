# Code of Conduct

This applies to issues, pull requests, discussions, and any other space
attached to the Vellum repository.

## How we treat each other

- Be kind. Disagree with the idea, not the person.
- No harassment, slurs, doxxing, sexualised content, or threats — direct
  or implied, in public or in DMs that started here.
- Assume the other person is reading on a bad day. Soften accordingly.
- "Vellum is opinionated" is a project value, not a licence to be a jerk
  in code review.
- English is the working language for issues and PRs, but it's a second
  language for many contributors. Don't grade grammar.

## What you contribute

- **Don't submit PRs that bundle vendor or trademarked icon packs** —
  AWS service icons, GCP, Azure, Atlassian, Slack, Cisco, anything in
  that category. The OSS core stays clean of trademarked third-party
  art for legal reasons. If you want to use those icons in your own
  diagrams, the in-app **+ Load** import flow is the supported path —
  you bring the licence, you take the responsibility.
- **Don't open issues asking for vendor icons to be bundled.** The
  answer is no, for the same reasons.
- Don't paste copyrighted code from another project into a PR without
  declaring it. PRs that look like a wholesale lift of another
  codebase will be closed.
- Don't open AI-generated PRs in volume. Single, focused, hand-reviewed
  changes are welcome regardless of how the diff was produced; bulk
  drive-by PRs are not.

## What you build with Vellum

Vellum is licensed under [PolyForm Noncommercial 1.0.0](./LICENSE).
As a community norm — in addition to and consistent with the licence:

- **Personal, hobby, educational, and internal-tool use is fine.** No
  permission needed.
- **Donation-supported community projects are fine.** Patreon, GitHub
  Sponsors, Open Collective, "buy me a coffee" — all OK, provided the
  project would still exist if the donations stopped. Donations fund
  the maintainer; they don't gate access to the work.
- **You cannot sell Vellum, charge for access to a hosted Vellum, or
  bundle Vellum into a paid product.** That includes paywalled SaaS,
  freemium tiers where Vellum sits behind the wall, license-key
  software, and "free for individuals, paid for teams" plays. If
  Vellum is on the revenue path, you'll need separate written
  permission from the copyright holder before shipping.
- A useful test: *if you removed the price tag, would the thing still
  work for the user?* If yes, donation-supported is fine. If no
  (paywalls, subscriptions, license keys, gated downloads), it's a
  paid product and out of scope for this licence.

If you're not sure which side of the line you're on, ask before
shipping. Honest questions get honest answers and won't trigger
enforcement.

## Enforcement

The maintainer ([Josh Morris](mailto:josh@blueprintr.io)) handles
reports. For behavioural issues or licensing questions, email with the
subject `[vellum coc]` and a link to what happened (or a description of
what you're planning). Security findings go to [SECURITY.md](./SECURITY.md),
not here.

Outcomes range from "this isn't a real issue, ignored" through private
warning, to PRs and issues from the account closed without review, to a
permanent block. There is no appeals committee — this is one person
applying judgement.

If the report is about the maintainer's behaviour, raise it publicly in
an issue, or fork the project. The licence permits the fork.
